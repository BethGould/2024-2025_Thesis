    \textbf{File:} Notebook\_BVP\_Plots.ipynb\\

\textbf{Problem:} Find the value of \(\psi^\prime_0\) which produces
equal initial and final values \(\psi\)

    This is my Jupyter notebook for finding values of \(\psi^\prime_0\)
which solve the BVP of the nonlinear Schrödinger equation for persistent
current in a single micrometer scale gold loop (\(M_{\mu}\)).

This file contains several plots and test scripts for solving the BVP of
our nonlinear solution. Since our equation for \(\psi^\prime_0\) now
depends on this value in a non-trivial way, we will proceed with a
numerical root finding algorithm to find solutions of \(\psi^\prime_0\)
which solve our BVP.

For our observed approximate solution to the nonlinear Schrödinger
equation \begin{equation}
\psi = e^{i M_{\mu} x} \left( a e^{i k_{\mu} x} + b e^{- i k_{\mu} x} \right),
\end{equation} we know that \(\psi(0) = \psi(2\pi R)\) when
\begin{equation}
\psi^\prime_0 = i M + k \left( \frac{e^{- 2 i M\pi R} - \cos\left( 2\pi k R \right)}{\sin\left(2\pi k R\right)}\right),
\end{equation}

where \(k\left(\mu, R, B, k, \psi^\prime_0 \right)\) and
\(M\left(\mu, R, B, k, \psi^\prime_0 \right)\) are given by our
previously solved model. Due to the \(\psi^\prime_0\) dependence of our
\(k\) and \(M\), we will proceed by moving \(\psi^\prime_0\) to the
right hand side and solving for the roots of the resulting expression.
The resulting \begin{equation}
F(\psi^\prime_0) =  \psi^\prime_0 - i M - k \left( \frac{e^{- 2 i M\pi R} - \cos\left( 2\pi k R \right)}{\sin\left(2\pi k R\right)}\right)
\end{equation} can be split into real and imaginary components as
follows: \begin{equation}
\mathrm{Re}\left(F\left(\psi^\prime_0\right)\right) =  \mathrm{Re}\left(\psi^\prime_0\right) - k \left( \frac{\cos\left(2 M\pi R\right) - \cos\left( 2\pi k R \right)}{\sin\left(2\pi k R\right)}\right)
\end{equation}

\begin{equation}
\mathrm{Im}\left(F\left(\psi^\prime_0\right)\right) = \mathrm{Im}\left(\psi^\prime_0\right) - M + k \left( \frac{\sin\left(2 M\pi R\right)}{\sin\left(2\pi k R\right)}\right)
\end{equation}

    Our BVP is solved when this function \(F(\psi^\prime_0) = 0\) (expressed
either as \(0+0i\) or \(\left[0,0\right]\)).

    \hypertarget{graphs-of-solutions-to-the-bvp}{%
\section{Graphs of Solutions to the
BVP}\label{graphs-of-solutions-to-the-bvp}}

    Now for some visualization of my function for BVP matching. We are
looking for roots of this function, which takes a complex number and
returns a complex number. To do this, we separate the real and imaginary
components of both the independent and dependent variables, and solve
with a multidimensional root finder. I will then hold either the real or
imaginary component of the independent variable (my \(\psi^\prime_0\))
constant, and plot both components of the dependent variable as separate
curves. Our roots occur when the intersection of both functions reaches
zero on both graphs. The fact that there are many solutions is visible
from the graphs, despite the fact that technically typically only one of
them is visible here. This is due to the fact that moving in the fixed
dimension moves the graph structure up or down, so any intersection
could become a root by adjusting the constant independent variable.
Calculating this effect is not straight forward, since the graphs don't
necessarily move together, but many (but not all) of the intersections
correspond to a root.

    Lets start with defining some functions, to reduce the amount of text
copied.

\section{3D Graphs of Solutions}\label{d-graphs-of-solutions}}

    These are my attempts at trying to produce a 3D plot of the solutions.
Something is visible, but they are not the greatest of quality. I
stopped trying to improve these as they are unnecessary for further
analysis. These plotting scripts tend to run slowly due to the large
number of points. I had tried to produce plots with plotly and seaborn
instead of matplotlib, but stopped due to time considerations, as better
graphics here were unnecessary.

    The plots use my function wrapper for the equation, whose roots give the
solution to the BVP. I will then wrap this function in another function,
which cuts off the values at a given limit. This is done because the
values rapidly increase, and if I don't limit them, the relevant part of
the function would not be visible on the graphs.

\section{Runs of BVP matching from the loop
class}\label{runs-of-bvp-matching-from-the-loop-class}}

    From the graphs above, it can be seen that we have a slight problem.
There are many solutions to the BVP and they have different current. I
have 4 solutions to the problem. 1. Add all the solutions together and
calculate the current from this. Strictly speaking the nonlinearity of
the solution means that this doesn't quite work (just like adding
solutions with different k values or considering that the overall
amplitude doesn't matter), but since it is weakly non-linear, this
difference may not be relevant. Finding all of the solutions, however,
is not such a simple task. There are also likely infinite solutions with
rapidly increasing current and amplitude. 2. Use the lowest energy
solution. This is most likely the correct solution, and the one we will
be using as it is the easiest to find. The solutions with the lowest
initial derivative are rare and easy to find as a unique solution. As I
increase my initial derivative, the current and amplitude rapidly
increase, as does the number of solutions. 3. Use the solution closest
to the linear solution. This has the logical benefit of considering
continuous diffeomorphism from the linear case. I will need to use low
\(\mu\) values for this to work, however. For greater \(\mu\) values,
figuring out which of the solutions is closest to the linear solution is
difficult. 4. Varying \(\mu\) based on \(|A_{max}|^2\). When I started
modeling the dependence of \(k\) on the parameters, I had thought that
something like this might be happening, i.e.~\(\mu\) is equal between
rings when \(|A_{max}|^2\) are equal, not when \(|A_0|^2\) are equal. I
still think this is logical, but \(\Delta k\) has a good Taylor
expansion for \(|A_0|^2\), not \(|A_{max}|^2\). This is what I found
when fitting the data. Adding this complication also most likely doesn't
help fix the problem with multiple solutions. Since it is still unclear
as to which variant of \(\mu\) matching is correct, I will use the
easier of the two variants for my analysis. The more difficult one just
adds another lengthy root-finding exercise.

    My analysis has shown that \(\mu\) of \(10^{-8}\) gives a stable
solution as long as the initial values don't grow too much, while still
showing a difference in current with the linear solution.

    The code below uses my find\_root\_many method within the loop class to
find multiple solutions to the BVP.
